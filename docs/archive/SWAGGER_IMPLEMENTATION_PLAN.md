## 📚 Library Selection

### Primary Choice: **swaggo/swag**

**Repository:** https://github.com/swaggo/swag

**Why swaggo/swag:**
- ✅ Most popular Go Swagger generator (8k+ stars)
- ✅ Built specifically for Gin framework
- ✅ Annotation-based (comments in code)
- ✅ Auto-generates from Go comments
- ✅ Supports OpenAPI 3.0
- ✅ Built-in Swagger UI
- ✅ Active maintenance

**Alternatives Considered:**
- `go-swagger`: More complex, overkill for our needs
- `kin-openapi`: Manual spec writing required

### Phase 4: Annotate Models (1 hour)

#### Step 4.1: APIResponse

```go
// APIResponse is the standard API response structure
// @Description Standard API response wrapper
type APIResponse struct {
    Status  bool        `json:"status" example:"true"`
    Message string      `json:"message" example:"Success"`
    Data    interface{} `json:"data,omitempty"`
    Error   *ErrorInfo  `json:"error,omitempty"`
}

// ErrorInfo contains detailed error information
// @Description Error details
type ErrorInfo struct {
    Code    string `json:"code" example:"ERROR_CODE"`
    Details string `json:"details" example:"Detailed error message"`
}
```

#### Step 4.2: Movie Model

```go
// Movie represents a movie from IDLIX featured list
// @Description Movie information from featured list
type Movie struct {
    URL    string `json:"url" example:"https://tv12.idlixku.com/movie/example-2024/"`
    Title  string `json:"title" example:"Example Movie (2024)"`
    Year   string `json:"year" example:"2024"`
    Type   string `json:"type" example:"movie"`
    Poster string `json:"poster" example:"https://image.tmdb.org/t/p/w185/poster.jpg"`
}
```

#### Step 4.3: VideoInfo Model

```go
// VideoInfo contains complete video information
// @Description Complete video information with streams and variants
type VideoInfo struct {
    VideoID           string            `json:"video_id" example:"163426"`
    VideoName         string            `json:"video_name" example:"Example Movie (2024)"`
    Poster            string            `json:"poster" example:"https://image.tmdb.org/..."`
    EmbedURL          string            `json:"embed_url" example:"https://jeniusplay.com/video/..."`
    M3U8URL           string            `json:"m3u8_url" example:"https://jeniusplay.com/cdn/hls/.../master.m3u8"`
    IsVariantPlaylist bool              `json:"is_variant_playlist" example:"true"`
    Variants          []VariantPlaylist `json:"variants,omitempty"`
    Subtitle          *SubtitleInfo     `json:"subtitle,omitempty"`
}

// VariantPlaylist represents a video quality variant
// @Description Video quality variant information
type VariantPlaylist struct {
    ID         string `json:"id" example:"0"`
    Resolution string `json:"resolution" example:"1920x1080"`
    Bandwidth  uint32 `json:"bandwidth" example:"1510000"`
    URI        string `json:"uri" example:"https://jeniusplay.com/hls/..."`
}

// SubtitleInfo contains subtitle information
// @Description Subtitle availability and URL
type SubtitleInfo struct {
    Available bool   `json:"available" example:"true"`
    URL       string `json:"url,omitempty" example:"https://jeniusplay.com/subs/...vtt"`
    Format    string `json:"format,omitempty" example:"vtt"`
}
```

## 📤 Export Options

### 1. Postman Collection

Convert `swagger.json` to Postman:
- Import in Postman: File → Import → swagger.json
- Auto-generates collection with all endpoints

### 2. OpenAPI Clients

Generate client SDKs:
```bash
# JavaScript/TypeScript
openapi-generator generate -i docs/swagger.yaml -g typescript-axios -o clients/typescript

# Python
openapi-generator generate -i docs/swagger.yaml -g python -o clients/python

# Java
openapi-generator generate -i docs/swagger.yaml -g java -o clients/java
```
